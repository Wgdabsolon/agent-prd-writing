# agent-prd-writing

为 AI Agent 产品编写 PRD 的专用方法论。传统 PRD 描述"页面和流程"，Agent PRD 描述"意图、判断和行动"。

## 核心理念

Agent 产品与传统产品的根本差异决定了 PRD 写法完全不同：

| 维度 | 传统产品 | Agent 产品 |
|------|----------|------------|
| 系统行为 | 可预期（点 A → 跳 B） | 不确定（持续理解 → 决策 → 行动） |
| 核心隐喻 | 自动售货机 | 聪明同事 |
| PRD 重点 | 页面结构、交互流程 | **意图识别、决策逻辑、边界控制** |
| 真正的"功能" | 按钮、页面 | **决策** |

本 Skill 提供一套完整的 Agent PRD 写作框架，涵盖：

- **意图空间拆解** — 从"用户说了什么"到"用户到底要完成什么"
- **工具调用规则** — 不是能力清单，是决策条件（该调才调，不该调别调）
- **边界条件控制** — 不是为了兜底，是为了控权（信息不足、工具失败、幻觉、越权）

## 功能特点

- 提供 7 步完整写作流程：场景定义 → 意图拆解 → 决策路径 → 工具调用规则 → 边界条件 → 结果定义 → 评估指标
- 内置完整的表格模板和 Mermaid 流程图示例
- 附带 AI 会议助手实战案例，展示意图→工具→边界的完整串联
- 自检清单确保 PRD 质量，避免产品设计遗漏

## 使用示例

### 触发方式

- **自动触发**：当对话涉及"Agent PRD"、"意图拆解"、"工具调用规则"、"边界条件"、"AI 产品需求"等关键词时，技能自动启用
- **手动触发**：在 Claude Code 技能列表中选择 `agent-prd-writing`，输入你的 Agent 产品需求要点

### 输入示例

```
我要做一个智能客服 Agent，需要写 PRD，帮我按照 Agent PRD 的方法论来拆解
```

或：

```
帮我设计一个会议纪要 Agent 的意图空间和工具调用规则
```

### 输出示例

技能会引导你按以下步骤完成 PRD：

1. **场景定义** — 明确用户画像、使用场景、痛点、为什么适合 Agent
2. **意图拆解** — 拆解核心意图（如：摘要型/行动项型/汇报型/沉淀型），填写意图结构表
3. **决策路径** — 生成判断流程图，标注工具调用触发点和用户确认点
4. **工具调用规则** — 为每种工具填写：触发条件、禁止条件、失败降级逻辑
5. **边界条件** — 逐项核对 8 类边界（信息不足、工具失败、权限不足、高风险动作等）
6. **结果定义** — 定义完成/部分完成/失败有中间结果三种状态
7. **评估指标** — 提供意图识别成功率、工具调用成功率、任务完成率等指标

最终输出一份结构完整的 Agent PRD 文档。
