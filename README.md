# agent-prd-writing

为 AI Agent 产品编写 PRD 的专用方法论。传统 PRD 描述"页面和流程"，Agent PRD 描述"意图、判断和行动"。

## 核心理念

Agent 产品与传统产品的根本差异决定了 PRD 写法完全不同：

| 维度 | 传统产品 | Agent 产品 |
|------|----------|------------|
| 系统行为 | 可预期（点 A → 跳 B） | 不确定（持续理解 → 决策 → 行动） |
| 核心隐喻 | 自动售货机 | 聪明同事 |
| PRD 重点 | 页面结构、交互流程 | **意图识别、决策逻辑、边界控制** |
| 真正的"功能" | 按钮、页面 | **决策** |

本 Skill 提供一套完整的 Agent PRD 写作框架，涵盖产品层和工程层两个维度。

## 功能特点

### 产品层（面向产品经理/业务方）
- **场景定义** — 伪 AI 需求过滤器，明确为什么用 Agent
- **意图空间拆解** — 从"用户说了什么"到"用户到底要完成什么"
- **决策路径设计** — 判断流程 / 工具触发点 / 用户确认点
- **工具调用规则** — 不是能力清单，是决策条件（该调才调，不该调别调）
- **边界条件控制** — 不是为了兜底，是为了控权（8 类边界条件）
- **结果定义与评估** — 完成状态 / 过程信息 / 评估指标

### 工程层（面向研发/测试/AI 工程师）
- **槽位定义（Slot Schema）** — 多轮对话的结构化参数定义（类型/校验/依赖）
- **System Prompt 规范** — Agent 核心行为的框架模板
- **工具 API 契约** — 每个工具的 Request/Response/错误码定义
- **支付流程设计** — 订单→支付→回调→出票→异常处理的完整闭环
- **退改政策定义** — 量化规则 + 配置化参数
- **数据表设计** — session / slot_state / decision_log / tool_log / transfer_queue
- **RAG Pipeline** — 分块策略 / 向量检索 / 混合检索 / 幻觉控制
- **MVP 范围界定** — 功能清单 / 排除范围 / 技术指标

## 使用示例

### 触发方式

- **自动触发**：当对话涉及"Agent PRD"、"意图拆解"、"工具调用规则"、"边界条件"、"AI 产品需求"等关键词时，技能自动启用
- **手动触发**：在 Claude Code 技能列表中选择 `agent-prd-writing`，输入你的 Agent 产品需求要点

### 输入示例

```
我要做一个智能客服 Agent，需要写 PRD，帮我按照 Agent PRD 的方法论来拆解
```

或：

```
帮我设计一个会议纪要 Agent 的意图空间和工具调用规则
```

### 输出示例

技能会引导你按以下步骤完成 PRD：

1. **场景定义** — 明确用户画像、使用场景、痛点、为什么适合 Agent
2. **意图拆解** — 拆解核心意图，填写意图结构表，定义优先级和冲突规则
3. **槽位定义** — 为每个多轮意图定义槽位（类型/必填/校验/依赖）
4. **决策路径** — 生成判断流程图，标注工具调用触发点和用户确认点
5. **工具调用规则** — 为每种工具填写：触发条件、禁止条件、失败降级逻辑、API 契约
6. **边界条件** — 逐项核对 8+ 类边界条件
7. **System Prompt** — 定义 Agent 行为框架
8. **支付流程**（如涉及交易）— 完整支付闭环
9. **结果定义** — 定义完成/部分完成/失败有中间结果三种状态
10. **评估指标** — 提供意图识别成功率、工具调用成功率等指标
11. **MVP 范围** — 明确首期功能和排除范围

最终输出一份结构完整的 Agent PRD 文档。
